# Mobile deployment guide

`export_mobile_runtime` freezes a CPU-eval module to TorchScript, reloads it, compares
logits exactly on the declared example, and writes a SHA-256 manifest. This is an export
smoke test, not a physical-mobile benchmark.

For Android, bundle the `.pt` file with PyTorch Android/Lite or translate the allow-listed
operator graph to ExecuTorch. Feed a contiguous `int64 [1, sequence]` tensor of UTF-8 byte
IDs and read `[1, sequence, 256]` next-byte logits. For iOS, use the equivalent PyTorch/
ExecuTorch runtime and the same byte contract. No tokenizer asset is required.

On every device record cold process start, model load, cake install/load, prompt prefill,
first output, steady decode p50/p95/p99, generated bytes/second, RSS/peak memory, package
storage, battery energy, and thermal state. Run the same locked prompts and hashes used on
desktop. Do not promote the mobile gate until at least one physical ARM Android or iOS run
is attached; absent hardware must remain `NOT_RUN_NO_HARDWARE`.
